## ⚠️ 当前存在的问题

### P2 - 低优先级（不影响使用）

#### 1. MCP 工具调用是占位实现

**文件**: `src/tools/MCPTool.ts`

**现状**:
```typescript
// 当前只是返回提示信息
return {
  success: true,
  content: `[MCP 调用] 服务器：${server}, 工具：${tool}...`,
};
```

**影响**: 
- 无法实际调用 MCP 服务器的工具
- 只能显示配置列表

**解决方案**:
- 安装 `@modelcontextprotocol/sdk`
- 实现 HTTP 客户端调用 MCP 协议
- 预计工作量：2-4 小时

**临时方案**: ✅ **使用 bash 直接调用 API**
```bash
# GitHub API 示例
❯ 用 bash 执行 "curl -H 'Authorization: token xxx' https://api.github.com/user"

# Sentry API 示例  
❯ 用 bash 执行 "curl -H 'Authorization: Bearer xxx' https://sentry.io/api/..."
```

**说明**: MCP 本身是连接外部服务的协议，如果只需要调用特定 API，直接用 bash + curl 更简单直接。只有在需要标准化、可扩展的工具系统时，才值得实现完整的 MCP 支持。

**用户决策**: ✅ 暂不实现完整 MCP，有需求时用 bash + curl 直接调用 API

---

#### 2. WebSearch 工具需要 API Key

**文件**: `src/tools/WebSearch.ts`

**现状**: 
- 依赖外部搜索 API（如 Brave Search）
- 未配置时无法使用

**影响**:
- 网络搜索功能不可用

**解决方案**:
- 配置 BRAVE_API_KEY 环境变量
- 或实现免费的搜索引擎爬虫
- 预计工作量：30min-1h

**临时方案**: ✅ **使用 bash 工具调用 `curl` 手动搜索**
```bash
# 示例：搜索 GitHub
❯ 用 bash 搜索 "curl https://api.github.com/search/repositories?q=claude+code"

# 示例：搜索网页
❯ 用 bash 搜索 "curl 'https://html.duckduckgo.com/html/?q=typescript+tutorial'"
```

**用户决策**: ✅ 当前使用 bash + curl 已足够，暂不实现专用 WebSearch 工具

### P3 - 建议优化（可选）

#### 4. 缺少任务完成语义理解

**问题**: 当前靠最大轮数（5 轮）强制结束对话

**现状**:
```typescript
const MAX_TURNS = 5;
for (let turn = 0; turn < MAX_TURNS; turn++) {
  // ...
}
```

**影响**:
- 复杂任务可能提前结束
- 简单任务可能浪费轮数

**解决方案**:
- 实现任务完成检测（LLM 判断）
- 动态调整最大轮数
- 预计工作量：1-2h

**当前状态**: 可用，但不完美

---
