# ✅ 使用说明改进

**改进时间**: 2026-04-07 01:52  
**问题**: AI 只创建任务，不执行实际开发

---

## 🐛 问题描述

**用户**: "开发一个坦克大战游戏"

**AI 响应**:
```
⏺ task_create_enhanced(title=坦克大战游戏开发)
 ⎿ ✅ 完成

[然后就直接结束了，没有真正开发游戏]
```

**问题**: AI 误解了用户意图，以为只是创建任务

---

## ✅ 解决方案

### 1. 明确表达需求

**不好的提示**:
```bash
"开发一个坦克大战游戏"
# AI 可能只创建任务
```

**好的提示**:
```bash
"开发一个坦克大战游戏，创建 Python 文件，包含玩家坦克、敌方坦克、地图和射击功能"
# AI 会真正创建文件
```

---

### 2. 使用明确动词

| 动词 | AI 理解 | 结果 |
|------|--------|------|
| "开发" | 可能创建任务 | ⚠️ 不确定 |
| "创建文件" | 明确创建文件 | ✅ 确定 |
| "编写代码" | 明确编写代码 | ✅ 确定 |
| "实现功能" | 明确实现功能 | ✅ 确定 |

---

### 3. 指定输出格式

**不明确**:
```bash
"开发坦克大战游戏"
```

**明确**:
```bash
"创建一个 Python 文件 tank_battle.py，实现坦克大战游戏，包含：
1. 玩家坦克控制
2. 敌方坦克 AI
3. 地图系统
4. 射击功能"
```

## 💡 最佳实践

### 1. 使用明确动词

```bash
# ✅ 好
"创建文件..."
"编写代码..."
"实现功能..."

# ❌ 不好
"开发..." (可能只创建任务)
```

---

### 2. 指定输出

```bash
# ✅ 好
"创建一个 Python 文件 tank.py，包含..."
"编写一个函数 calculate()，实现..."

# ❌ 不好
"做一个坦克游戏"
```

---

### 3. 分步骤说明

```bash
"创建坦克大战游戏，需要：
1. 创建 tank_battle.py 文件
2. 实现玩家坦克类
3. 实现敌方坦克类
4. 实现地图系统
5. 实现射击功能"
```

## ✅ 总结

**问题原因**: AI 误解用户意图，以为只是创建任务

**解决方案**:
1. ✅ 使用明确动词（创建、编写、实现）
2. ✅ 指定输出格式（文件、代码）
3. ✅ 分步骤说明需求

**最佳实践**:
```bash
# ✅ 推荐
"创建文件 X，实现功能 Y，包含 A、B、C"

# ❌ 避免
"开发 X"
```

**现在 AI 会正确理解并执行实际开发了！** 🎉
